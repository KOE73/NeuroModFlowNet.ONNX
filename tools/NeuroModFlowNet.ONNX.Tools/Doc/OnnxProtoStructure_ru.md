Для понимания общей архитектуры ONNX важно представлять её как матрешку: один класс содержит списки других, и всё это в итоге собирается в единый граф.
Вот описание ключевых классов (включая те, что ты уже знаешь), которые формируют скелет модели:
1. ModelProto (Верхний уровень)
Это корень всего файла. Он не содержит логики вычислений, но хранит всё остальное.
•	Значение: Контейнер для графа, метаданных и информации о версиях.
•	Связи: Содержит один GraphProto и список StringStringEntryProto (MetadataProps).
2. GraphProto (Сердце модели)
Описывает топологию нейросети.
•	Значение: Определяет, как данные текут от входа к выходу.
•	Связи: Содержит списки NodeProto, ValueInfoProto (входы/выходы) и TensorProto (веса).
3. NodeProto (Оператор)
Конкретный шаг вычислений (например, Conv, Add, Relu).
•	Значение: Описывает операцию. Связывается с другими узлами не напрямую, а через имена строк в списках input и output.
•	Связи: Содержит список AttributeProto для настройки параметров (например, stride или padding).
4. AttributeProto (Настройки узла)
•	Значение: Хранит именованные параметры для NodeProto.
•	Важно: Атрибут может быть числом, строкой, тензором или даже другим графом (для циклов или условий).
5. ValueInfoProto (Описание тензоров)
Описывает "порты", через которые проходят данные.
•	Значение: Хранит имя тензора, его тип (например, float32) и форму (shape, например [1, 3, 640, 640]).
•	Связи: Используется в GraphProto для описания input и output всей модели.
6. TensorProto (Инициализатор / Веса)
Это статические данные, которые не меняются во время работы (веса сверток, смещения).
•	Значение: Хранит сами массивы чисел.
•	Связи: Находится в списке initializer внутри GraphProto. Если имя инициализатора совпадает с именем входа узла, узел берет данные отсюда, а не от предыдущего слоя.
7. TypeProto (Система типов)
Сложная структура внутри ValueInfoProto.
•	Значение: Определяет, является ли объект тензором, последовательностью (Sequence) или картой (Map).
•	Связи: Содержит Tensor, который в свою очередь содержит elem_type и ShapeProto.
8. TensorShapeProto (Форма данных)
•	Значение: Список размерностей. Каждая размерность может быть числом или строковым параметром (символьная размерность для динамических батчей).

Визуальная иерархия и взаимосвязи
Граф строится на сопоставлении строк. Если Node A имеет output = ["tensor_x"], а Node B имеет input = ["tensor_x"], они считаются связанными.
Класс	Где живет	Что внутри (основное)
ModelProto	Корень	graph, metadata_props
GraphProto	в Model	node (список), initializer (веса), input/output
NodeProto	в Graph	op_type, input (имена), output (имена), attribute
TensorProto	в Graph	raw_data или список чисел, dims
ValueInfoProto	в Graph	name, type (тип и shape)
